- `GardenaDevice`: Device data structure

  - Device discovery
  - Device control
  - WebSocket connection
- `GardenaPlatform`: Platform class
  - Lifecycle management
  - Device registration
  - Command handling
- `initializePlugin()`: Plugin entry point

**Key Features**:

- Full TypeScript implementation
- Comprehensive error handling
- Detailed logging
- JSDoc documentation

## Schema Files

### üìã gardena.schema.json

**Configuration validation schema**

- JSON Schema format
- Property definitions
- Type validation
- Required fields
- Examples

**Properties**:

- `name`: Plugin name
- `type`: Platform type (DynamicPlatform)
- `apiKey`: Gardena API key (required)
- `email`: Account email (optional)
- `password`: Account password (optional)
- `systemId`: System ID (optional)
- `debug`: Debug mode flag
- `unregisterOnShutdown`: Cleanup flag

```bash
npm run build          # Compile TypeScript
npm run watch          # Watch mode compilation
npm run clean          # Clean build artifacts
npm run cleanBuild     # Clean and rebuild
```

### Quality Assurance

```bash
npm run lint           # Check code quality
```
